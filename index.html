What is a Unix timestamp?

A Unix timestamp (also called epoch time) is the number of seconds that have elapsed since January 1, 1970 at 00:00:00 UTC. It's a universal way to represent a point in time used across programming languages and systems.

What is the current Unix timestamp?

The live timestamp is shown at the top of this page — it updates every second. In JavaScript you can get it with Math.floor(Date.now() / 1000), in Python with import time; time.time().

What is the difference between seconds and milliseconds timestamps?

Unix timestamps are traditionally in seconds. JavaScript's Date.now() returns milliseconds (1000× larger). Always check which format an API expects — dividing a millisecond timestamp by 1000 gives you seconds.

What is the Unix timestamp for January 1, 2000?

January 1, 2000 00:00:00 UTC = Unix timestamp 946684800. This is also known as Y2K — it passed without the apocalypse.

When does Unix time overflow (Year 2038 problem)?

32-bit signed integers can store Unix timestamps up to 2147483647, which corresponds to January 19, 2038 at 03:14:07 UTC. After this, 32-bit systems will overflow. Most modern systems use 64-bit integers which won't overflow for billions of years.
